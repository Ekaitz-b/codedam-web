    Entornos de Desarrollo Integrado (IDEs)

    Un IDE es una aplicación que proporciona herramientas completas para el desarrollo de software.

    IDEs más Populares para Java

    • IntelliJ IDEA: El más popular y potente. Community Edition gratuita
    • Eclipse: Código abierto, muy configurable, ampliamente usado
    • NetBeans: Oficial de Oracle, excelente para principiantes
    • Visual Studio Code: Ligero, con extensiones para Java

    Características Clave de un IDE

    • Editor de código con syntax highlighting
    • Autocompletado inteligente (IntelliSense)
    • Depurador (debugger) integrado
    • Refactorización automática
    • Integración con sistemas de control de versiones
    • Gestión de dependencias (Maven, Gradle)
    • Análisis estático de código

    Control de Versiones con Git

    Git es el sistema de control de versiones más utilizado en el mundo del desarrollo.

    Conceptos Básicos

    • Repositorio: Carpeta que contiene tu proyecto y su historial
    • Commit: Instantánea del proyecto en un momento dado
    • Branch: Rama paralela de desarrollo
    • Merge: Fusionar cambios de una rama a otra
    • Remote: Repositorio remoto (GitHub, GitLab)

    Comandos Esenciales Git

    # Configuración inicial git config --global user.name "Tu Nombre" git config --global user.email "tu@email.com" # Crear repositorio git init git clone https://github.com/usuario/repo.git # Estados y commits git status # Ver estado git add archivo.java # Añadir archivo al staging git add . # Añadir todos los archivos git commit -m "Mensaje" # Hacer commit # Historial git log # Ver historial git log --oneline # Historial resumido git diff # Ver cambios # Ramas (branches) git branch # Listar ramas git branch nueva-feature # Crear rama git checkout nueva-feature # Cambiar a rama git checkout -b bug-fix # Crear y cambiar git merge nueva-feature # Fusionar rama # Repositorio remoto git remote add origin URL git push -u origin main # Subir cambios git pull # Descargar cambios git fetch # Descargar sin fusionar

    Flujo de Trabajo Git

    # 1. Clonar repositorio git clone https://github.com/equipo/proyecto.git # 2. Crear rama para nueva funcionalidad git checkout -b feature/login # 3. Hacer cambios y commits git add . git commit -m "Añadir formulario de login" # 4. Subir rama al remoto git push -u origin feature/login # 5. Crear Pull Request en GitHub # 6. Después de aprobar, fusionar con main git checkout main git merge feature/login git push

    Buenas prácticas: Commits pequeños y frecuentes, mensajes descriptivos, nunca hacer commit de contraseñas o secretos.

    Testing y Pruebas

    Las pruebas aseguran que el código funciona correctamente y facilitan el mantenimiento.

    Tipos de Pruebas

    • Pruebas Unitarias: Prueban unidades individuales (métodos, funciones)
    • Pruebas de Integración: Prueban la interacción entre componentes
    • Pruebas de Sistema: Prueban el sistema completo
    • Pruebas de Aceptación: Verifican requisitos del cliente

    JUnit - Testing en Java

    import org.junit.jupiter.api.Test; import static org.junit.jupiter.api.Assertions.*; public class CalculadoraTest { @Test public void testSumar() { Calculadora calc = new Calculadora(); int resultado = calc.sumar(5, 3); assertEquals(8, resultado, "5 + 3 debe ser 8"); } @Test public void testDividir() { Calculadora calc = new Calculadora(); double resultado = calc.dividir(10, 2); assertEquals(5.0, resultado, 0.001); } @Test public void testDividirPorCero() { Calculadora calc = new Calculadora(); assertThrows(ArithmeticException.class, () -> { calc.dividir(10, 0); }); } @Test public void testMultiplicarNegativo() { Calculadora calc = new Calculadora(); assertTrue(calc.multiplicar(-2, 3) < 0); } }

    TDD - Test Driven Development

    Metodología donde escribes las pruebas ANTES que el código:

    1. Escribir prueba (falla porque no hay código)
    2. Escribir código mínimo para pasar la prueba
    3. Refactorizar el código
    4. Repetir

    Debugging - Depuración

    El debugging es el proceso de encontrar y corregir errores en el código.

    Técnicas de Depuración

    • Breakpoints: Pausar ejecución en una línea específica
    • Step Over: Ejecutar línea actual sin entrar en métodos
    • Step Into: Entrar dentro de un método
    • Step Out: Salir del método actual
    • Watch Variables: Observar valores de variables
    • Evaluate Expression: Evaluar expresiones en tiempo de ejecución

    Errores comunes: NullPointerException (objeto null), ArrayIndexOutOfBoundsException (índice fuera de rango), StackOverflowError (recursión infinita).

    Metodologías Ágiles

    Scrum

    Framework ágil más popular para gestión de proyectos.

    • Sprint: Iteración de 1-4 semanas
    • Product Backlog: Lista priorizada de tareas
    • Daily Standup: Reunión diaria de 15 min
    • Sprint Review: Demostración al final del sprint
    • Sprint Retrospective: Reflexión sobre el proceso

    Kanban

    Sistema visual de gestión de flujo de trabajo.

    • Tablero con columnas: To Do, In Progress, Done
    • Límite de tareas en progreso (WIP)
    • Flujo continuo sin sprints fijos

    Herramientas de Gestión

    • Jira: Popular para Scrum y Kanban
    • Trello: Simple y visual, ideal para Kanban
    • GitHub Projects: Integrado con repositorios
    • Asana: Gestión de tareas y proyectos

    Gestión de Dependencias

    Maven

    <!-- pom.xml --> <project> <modelVersion>4.0.0</modelVersion> <groupId>com.codedam</groupId> <artifactId>mi-proyecto</artifactId> <version>1.0-SNAPSHOT</version> <dependencies> <dependency> <groupId>org.junit.jupiter</groupId> <artifactId>junit-jupiter</artifactId> <version>5.9.0</version> <scope>test</scope> </dependency> </dependencies> </project>

    Gradle

    // build.gradle plugins { id 'java' } group 'com.codedam' version '1.0-SNAPSHOT' repositories { mavenCentral() } dependencies { testImplementation 'org.junit.jupiter:junit-jupiter:5.9.0' }
