    Introducción a Lenguajes de Marcas

    Los lenguajes de marcas son sistemas de codificación que utilizan etiquetas (marcas) para estructurar y dar formato a documentos. A diferencia de los lenguajes de programación, los lenguajes de marcas se centran en la presentación y estructura del contenido.

    ¿Qué son los Lenguajes de Marcas?

    Un lenguaje de marcas usa etiquetas (tags) para marcar diferentes partes de un documento. Las etiquetas no son visibles al usuario final, pero indican cómo debe procesarse el contenido.

    • HTML (HyperText Markup Language): Estructura de páginas web
    • XML (eXtensible Markup Language): Almacenamiento y transporte de datos
    • SVG (Scalable Vector Graphics): Gráficos vectoriales
    • Markdown: Formato de texto simple (este mismo README)

    Evolución de los Lenguajes Web

    La web ha evolucionado desde páginas estáticas simples hasta aplicaciones complejas:

    • 1991 - HTML 1.0: Tim Berners-Lee crea la primera versión
    • 1996 - CSS 1: Se separa presentación de contenido
    • 1998 - XML 1.0: W3C estandariza XML
    • 2000 - XHTML: HTML reformulado como XML
    • 2006 - JSON: Formato ligero para intercambio de datos
    • 2014 - HTML5: Estándar moderno con APIs multimedia
    • 2023 - CSS Grid/Flexbox: Layouts modernos sin frameworks

    HTML5 - HyperText Markup Language

    ¿Qué es HTML5?

    HTML5 es la quinta y actual versión del estándar HTML. Define la estructura y semántica del contenido web. HTML5 introdujo nuevas etiquetas semánticas, APIs multimedia, almacenamiento local, y eliminó elementos obsoletos como <font> y <center>.

    Estructura Básica de un Documento HTML5

    Plantilla HTML5 Completa 
    <!DOCTYPE html>  <!-- Declaración de HTML5 (OBLIGATORIA) -->
<html lang="es">  <!-- Idioma del documento para accesibilidad -->
<head>
    <!-- Metadatos del documento (NO visibles) -->

    <!-- Codificación de caracteres (UTF-8 soporta todos los idiomas) -->
    <meta charset="UTF-8">

    <!-- Responsive: viewport para dispositivos móviles -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0">

    <!-- SEO: Descripción que aparece en resultados de Google -->
    <meta name="description" content="Aprende HTML5 desde cero con CodeDAM">

    <!-- SEO: Palabras clave (menos importante hoy en día) -->
    <meta name="keywords" content="HTML5, tutorial, programación, web">

    <!-- SEO: Autor del documento -->
    <meta name="author" content="CodeDAM">

    <!-- Título de la pestaña del navegador (MUY IMPORTANTE para SEO) -->
    <title>Mi Primera Página Web - CodeDAM</title>

    <!-- Enlace a hoja de estilos CSS externa -->
    <link rel="stylesheet" href="css/styles.css">

    <!-- Favicon (icono de la pestaña) -->
    <link rel="icon" href="images/favicon.ico" type="image/x-icon">

    <!-- Open Graph para redes sociales (Facebook, LinkedIn) -->
    <meta property="og:title" content="Mi Primera Página Web">
    <meta property="og:description" content="Tutorial completo de HTML5">
    <meta property="og:image" content="https://ejemplo.com/imagen.jpg">

    <!-- Twitter Cards -->
    <meta name="twitter:card" content="summary_large_image">
</head>

<body>
    <!-- Contenido visible de la página -->

    <header>
        <h1>Bienvenido a Mi Sitio Web</h1>
    </header>

    <main>
        <section>
            <h2>Contenido Principal</h2>
            <p>Este es mi primer párrafo en HTML5.</p>
        </section>
    </main>

    <footer>
        <p>&copy; 2025 CodeDAM. Todos los derechos reservados.</p>
    </footer>

    <!-- JavaScript al final del body para mejor rendimiento -->
    <script src="js/script.js"></script>
</body>
</html>

    Etiquetas Semánticas HTML5

    HTML5 introdujo etiquetas semánticas que describen el significado del contenido, no solo su presentación. Esto mejora SEO, accesibilidad y la mantenibilidad del código.

    Atributos Importantes de Formularios 
    <!-- ATRIBUTOS DE VALIDACIÓN HTML5 -->

<!-- required - Campo obligatorio -->
<input type="text" name="nombre" required>

<!-- minlength / maxlength - Longitud mínima/máxima -->
<input type="text" name="username" minlength="3" maxlength="20">

<!-- min / max - Valor mínimo/máximo (para number, date, etc.) -->
<input type="number" name="edad" min="18" max="100">
<input type="date" name="fecha" min="2024-01-01" max="2024-12-31">

<!-- step - Incremento permitido -->
<input type="number" name="precio" min="0" step="0.01">  <!-- Permite decimales -->

<!-- pattern - Expresión regular para validación -->
<input type="text" name="dni" pattern="[0-9]{8}[A-Z]" title="8 dígitos + 1 letra mayúscula">

<!-- placeholder - Texto de ayuda (desaparece al escribir) -->
<input type="email" name="email" placeholder="tu@email.com">

<!-- autocomplete - Sugerencias del navegador -->
<input type="email" name="email" autocomplete="email">
<input type="text" name="nombre" autocomplete="name">

<!-- readonly - Solo lectura (se envía el valor) -->
<input type="text" name="id" value="12345" readonly>

<!-- disabled - Deshabilitado (NO se envía el valor) -->
<input type="text" name="campo" value="No editable" disabled>

<!-- autofocus - Foco automático al cargar página (solo uno) -->
<input type="text" name="buscar" autofocus>

<!-- multiple - Permite selección/subida múltiple -->
<input type="file" name="archivos" multiple>
<select name="opciones" multiple>
    <option>Opción 1</option>

</select>

    ⚠️ Errores Comunes en HTML5
    • Olvidar cerrar etiquetas: <div>contenido ❌ → <div>contenido</div> ✅
    • IDs duplicados: Solo puede haber UN elemento con cada ID por página
    • Anidar mal las etiquetas: <p><div></p></div> ❌ → <div><p></p></div> ✅
    • No usar alt en imágenes: Malo para accesibilidad y SEO
    • Usar <br> para espaciado: Usa CSS (margin, padding)
    • Múltiples <main> en una página: Solo puede haber uno
    • Formularios sin <label>: Malo para accesibilidad
    • Tablas para layout: Usa CSS Grid/Flexbox
    • No especificar type en botones: Por defecto es submit (puede enviar formulario sin querer)
    💡 Pro Tips - HTML5
    • Usa HTML semántico siempre: mejora SEO y accesibilidad automáticamente
    • <main>, <header>, <footer> facilitan navegación con teclado
    • Valida tu HTML con W3C Validator
    • Indenta correctamente: 2 o 4 espacios consistentes (no tabs y espacios mezclados)
    • Usa loading="lazy" en imágenes para mejor rendimiento
    • Open Graph meta tags mejoran apariencia en redes sociales
    • Attribute autocomplete ayuda al usuario (y a password managers)
    • No uses autoplay en videos/audio (molesta usuarios)
    • Escribe alt descriptivo, no solo "imagen" (piensa en usuarios ciegos)
    • Usa <details> y <summary> para FAQs (acordeones nativos)
    • Mobile First: Diseña primero para móvil, luego adapta a desktop

    Box Model (Modelo de Caja)

    El Box Model es fundamental en CSS. Cada elemento HTML es una "caja" con: content (contenido), padding (relleno interno), border (borde) y margin (margen externo).

    Box Model Explicado Visualmente
    /* ESTRUCTURA DEL BOX MODEL:
 *
 *  ┌─────────────────────────────────────┐
 *  │           MARGIN (exterior)         │
 *  │  ┌───────────────────────────────┐  │
 *  │  │      BORDER (borde)           │  │
 *  │  │  ┌─────────────────────────┐  │  │
 *  │  │  │   PADDING (relleno)     │  │  │
 *  │  │  │  ┌───────────────────┐  │  │  │
 *  │  │  │  │    CONTENT        │  │  │  │
 *  │  │  │  │   (contenido)     │  │  │  │
 *  │  │  │  └───────────────────┘  │  │  │
 *  │  │  └─────────────────────────┘  │  │
 *  │  └───────────────────────────────┘  │
 *  └─────────────────────────────────────┘
 */

.caja-ejemplo {
    /* Content (contenido) */
    width: 300px;
    height: 200px;

    /* Padding (relleno interno - espacio entre contenido y borde) */
    padding: 20px;  /* Todos los lados */
    /* padding: 10px 20px; */  /* vertical horizontal */
    /* padding: 10px 20px 15px 25px; */  /* top right bottom left (sentido horario) */

    /* Border (borde) */
    border: 2px solid #667eea;
    border-radius: 8px;  /* Esquinas redondeadas */

    /* Margin (margen externo - espacio entre esta caja y otras) */
    margin: 20px;  /* Todos los lados */
    /* margin: 10px auto; */  /* vertical=10px, horizontal=centrado */

    /* Background */
    background-color: #f0f0f0;
}

    Posicionamiento CSS

    La propiedad position controla cómo se ubican los elementos en la página.

    Tipos de Posicionamiento
    /* 1. POSITION: STATIC (por defecto) */
.elemento-normal {
    position: static;  /* Flujo normal del documento */
    /* No responde a top, right, bottom, left */
}

/* 2. POSITION: RELATIVE */
/* Se posiciona RELATIVO a su posición original en el flujo */
.elemento-relativo {
    position: relative;
    top: 20px;     /* Se mueve 20px HACIA ABAJO desde su posición original */
    left: 30px;    /* Se mueve 30px HACIA LA DERECHA */
    /* El espacio original se mantiene reservado */
}

/* Uso común: Crear contexto para position absolute */
.contenedor-padre {
    position: relative;  /* Punto de referencia para hijos absolute */
}

/* 3. POSITION: ABSOLUTE */
/* Se posiciona RELATIVO al ancestro con position (relative/absolute/fixed) más cercano */
.elemento-absoluto {
    position: absolute;
    top: 0;        /* 0px desde el top del contenedor padre */
    right: 0;      /* 0px desde el right del contenedor padre */
    /* Se SACA del flujo normal (no ocupa espacio) */
}

/* Ejemplo práctico: Badge en esquina */
.tarjeta {
    position: relative;
    width: 300px;
    height: 200px;
    border: 1px solid #ddd;
}

.badge {
    position: absolute;
    top: 10px;
    right: 10px;
    background-color: red;
    color: white;
    padding: 5px 10px;
    border-radius: 15px;
}

.centrado-absoluto {
    position: absolute;
    top: 50%;
    left: 50%;

    Flexbox - Layout Flexible

    Flexbox es un sistema de layout unidimensional (fila o columna) ideal para distribuir espacio y alinear elementos.

    Flexbox - Propiedades del Contenedor
    /* ACTIVAR FLEXBOX */
.contenedor-flex {
    display: flex;  /* Activa Flexbox */
    /* Los hijos directos se convierten en "flex items" */
}

/* DIRECCIÓN DEL EJE PRINCIPAL */
.flex-fila {
    display: flex;
    flex-direction: row;  /* Por defecto - horizontal → */
}

.flex-fila-inversa {
    display: flex;
    flex-direction: row-reverse;  /* Horizontal invertido ← */
}

.flex-columna {
    display: flex;
    flex-direction: column;  /* Vertical ↓ */
}

.flex-columna-inversa {
    display: flex;
    flex-direction: column-reverse;  /* Vertical invertido ↑ */
}

/* ENVOLVER ELEMENTOS (WRAP) */
.flex-wrap {
    display: flex;
    flex-wrap: wrap;  /* Los elementos saltan a nueva línea si no caben */
}

.flex-nowrap {
    display: flex;
    flex-wrap: nowrap;  /* Por defecto - todos en una línea, se encogen */
}

.flex-wrap-reverse {
    display: flex;
    flex-wrap: wrap-reverse;  /* Envuelve en dirección inversa */
}

/* ATAJO: flex-flow = direction + wrap */
.flex-combinado {
    display: flex;
    flex-flow: row wrap;  /* Equivale a: flex-direction: row; flex-wrap: wrap; */
}

/* JUSTIFICACIÓN - Eje Principal (horizontal si row) */
.justify-start {
    display: flex;
    justify-content: flex-start;  /* Por defecto - al inicio ←| */
}

.justify-end {
    display: flex;
    justify-content: flex-end;  /* Al final |→ */
}

.justify-center {
    display: flex;
    justify-content: center;  /* Centrado |←→| */
}

.justify-space-between {
    display: flex;
    justify-content: space-between;  /* |←···→···←| Espacio entre elementos */
}

.justify-space-around {
    display: flex;
    justify-content: space-around;  /* ·|←···→···←|· Espacio alrededor */
}

.justify-space-evenly {
    display: flex;
    justify-content: space-evenly;  /* ···|←···→···←|··· Espacio uniforme */
}

/* ALINEACIÓN - Eje Secundario (vertical si row) */
.align-start {
    display: flex;
    align-items: flex-start;  /* Arriba (si row) */

    ❌ Errores Comunes en CSS

    • Olvidar box-sizing: border-box → Anchos calculados incorrectamente (300px width + 20px padding = 340px total)
    • No usar unidades relativas (rem, em, %) → Diseño no escalable ni accesible
    • Abusar de !important → Hace el CSS imposible de mantener, solo usa en casos extremos
    • Position absolute sin contenedor relative → El elemento se posiciona respecto al <body>
    • Z-index sin position → z-index no funciona en elementos static
    • Flexbox para layouts 2D complejos → Usa Grid para layouts bidimensionales
    • Olvidar vendor prefixes en propiedades nuevas → Usa autoprefixer o añade -webkit-, -moz-, etc.
    • No probar en diferentes navegadores → Cada navegador interpreta CSS ligeramente diferente
    • Selectores demasiado específicos → #header .nav ul li a → Difícil de sobrescribir
    • Animaciones sin considerar prefers-reduced-motion → Accesibilidad para usuarios con mareos
    • No usar CSS custom properties (variables) → Repetir colores y valores dificulta mantenimiento
    • Olvidar el viewport meta tag → El responsive design NO funcionará en móviles

    💡 Consejos Pro CSS

    • Mobile First: Diseña primero para móvil, luego usa min-width media queries para desktop
    • Variables CSS: :root { --color-primary: #667eea; } y color: var(--color-primary);
    • Metodología BEM: .bloque__elemento--modificador para nombres de clases consistentes
    • CSS Grid + Flexbox: Usa Grid para layouts generales, Flexbox para componentes dentro
    • clamp() para tipografía responsive: font-size: clamp(16px, 4vw, 32px);
    • aspect-ratio: aspect-ratio: 16/9; para mantener proporciones sin padding-hack
    • Usa rem para tamaños de fuente: Escalable con zoom del navegador (accesibilidad)
    • DevTools son tus amigos: F12 → Inspector → Prueba CSS en tiempo real
    • object-fit: cover para imágenes: Mantiene proporciones en contenedores fijos
    • gap en Flexbox/Grid: Más limpio que margin en cada elemento
    • Organiza tu CSS: Variables → Reset → Layout → Componentes → Utilities → Media queries
    • Aprende Sass/SCSS: Variables, nesting, mixins hacen el CSS más mantenible
    • CSS Grid DevTools: Firefox tiene las mejores herramientas para debuggear Grid
    • Prueba CanIUse.com: Verifica compatibilidad de propiedades CSS antes de usar

    XML - eXtensible Markup Language

    Introducción a XML

    XML (eXtensible Markup Language) es un lenguaje de marcado diseñado para almacenar y transportar datos de forma estructurada, legible tanto para humanos como para máquinas. A diferencia de HTML que se enfoca en la presentación, XML se enfoca en la descripción de datos.

    Características de XML

    • Extensible: Puedes crear tus propias etiquetas personalizadas
    • Auto-descriptivo: Los datos se describen a sí mismos mediante etiquetas significativas
    • Independiente de plataforma: Texto plano que funciona en cualquier sistema
    • Estándar W3C: Mantenido por el World Wide Web Consortium
    • Jerárquico: Estructura de árbol con raíz, padres e hijos

    XML vs HTML

    Diferencias Clave
    <!-- HTML - Enfocado en PRESENTACIÓN -->
<h1>Juan Pérez</h1>
<p>Edad: 25</p>
<p>Email: juan@email.com</p>

<!-- XML - Enfocado en DESCRIPCIÓN de DATOS -->
<persona>
    <nombre>Juan Pérez</nombre>
    <edad>25</edad>
    <email>juan@email.com</email>
</persona>

<!-- ═══════════════════════════════════════════ -->
<!-- DIFERENCIAS PRINCIPALES -->
<!-- ═══════════════════════════════════════════ -->

HTML:
  ✓ Etiquetas predefinidas (<h1>, <p>, <div>)
  ✓ Enfoque en visualización
  ✓ Más permisivo con errores
  ✗ No permite crear etiquetas propias

XML:
  ✓ Etiquetas personalizadas (<persona>, <producto>, <pedido>)
  ✓ Enfoque en almacenamiento de datos
  ✓ Estricto (debe estar bien formado)
  ✓ Independiente de la presentación

    Reglas que DEBE Cumplir Todo XML
    <?xml version="1.0" encoding="UTF-8"?>

<!-- ═══════════════════════════════════════════════════════ -->
<!-- REGLA 1: Debe haber UN SOLO elemento raíz -->
<!-- ═══════════════════════════════════════════════════════ -->

<catalogo>

<!-- REGLA 2: Todas las etiquetas deben cerrarse -->

<!-- ✅ CORRECTO -->
<nombre>Juan</nombre>
<edad>25</edad>
<vacio/>  <!-- Etiqueta vacía auto-cerrada -->

<!-- ❌ INCORRECTO (en XML, a diferencia de HTML) -->
<nombre>Juan
<edad>25


<!-- ═══════════════════════════════════════════════════════ -->
<!-- REGLA 3: Las etiquetas son CASE-SENSITIVE -->
<!-- ═══════════════════════════════════════════════════════ -->

<!-- ✅ CORRECTO -->
<Nombre>Juan</Nombre>

<!-- ❌ INCORRECTO - Mayúsculas diferentes -->
<Nombre>Juan</nombre>  <!-- "Nombre" ≠ "nombre" -->


<!-- ═══════════════════════════════════════════════════════ -->
<!-- REGLA 4: Anidación correcta (no se pueden cruzar) -->
<!-- ═══════════════════════════════════════════════════════ -->

<!-- ✅ CORRECTO - Anidación correcta -->
<persona>
    <nombre>Juan</nombre>
    <direccion>
        <calle>Gran Vía</calle>
        <ciudad>Madrid</ciudad>
    </direccion>
</persona>

<!-- ❌ INCORRECTO - Etiquetas cruzadas -->
<persona>
    <nombre>Juan</persona>
</nombre>


<!-- ═══════════════════════════════════════════════════════ -->
<!-- REGLA 5: Valores de atributos SIEMPRE entre comillas -->
<!-- ═══════════════════════════════════════════════════════ -->

<!-- ✅ CORRECTO -->
<libro idioma="es" paginas="350">...</libro>

<!-- ❌ INCORRECTO - Sin comillas -->
<libro idioma=es paginas=350>...</libro>


<!-- ═══════════════════════════════════════════════════════ -->
<!-- REGLA 6: Atributos únicos por elemento -->
<!-- ═══════════════════════════════════════════════════════ -->

<!-- ✅ CORRECTO -->
<producto id="123" nombre="Portátil">...</producto>

<!-- ❌ INCORRECTO - Atributo duplicado -->
<producto id="123" id="456">...</producto>


<!-- ═══════════════════════════════════════════════════════ -->
<!-- REGLA 7: Caracteres especiales deben escaparse -->
<!-- ═══════════════════════════════════════════════════════ -->

<!-- ✅ CORRECTO - Caracteres escapados -->
<expresion>3 &lt; 5 &amp;&amp; 10 &gt; 8</expresion>
<!-- Resultado: 3 < 5 && 10 > 8 -->

<!-- Entidades predefinidas en XML: -->
<!-- &lt;   = < (less than) -->
<!-- &gt;   = > (greater than) -->
<!-- &amp;  = & (ampersand) -->
<!-- &apos; = ' (apostrophe) -->
<!-- &quot; = " (quotation) -->

<!-- ❌ INCORRECTO - < y & sin escapar -->
<expresion>3 < 5 & 10 > 8</expresion>

    Espacios de Nombres (Namespaces)

    Los namespaces permiten evitar conflictos de nombres cuando se combinan vocabularios XML de diferentes fuentes.

    Problema: Conflicto de Nombres
    <?xml version="1.0" encoding="UTF-8"?>

<!-- PROBLEMA: Dos elementos "tabla" con significados diferentes -->

<documento>
    <!-- "tabla" como mueble -->
    <tabla>
        <material>madera</material>
        <patas>4</patas>
    </tabla>

    <!-- "tabla" como tabla HTML -->
    <tabla>
        <fila>
            <celda>Dato 1</celda>
            <celda>Dato 2</celda>
        </fila>
    </tabla>

    <!-- ❌ ¡CONFLICTO! Dos elementos con el mismo nombre pero significado diferente -->
</documento>
    Solución: Usar Namespaces
    <?xml version="1.0" encoding="UTF-8"?>

<!-- ═══════════════════════════════════════════════════════ -->
<!-- SOLUCIÓN: Declarar namespaces con prefijos -->
<!-- ═══════════════════════════════════════════════════════ -->

<documento
    xmlns:mueble="http://www.ejemplo.com/muebles"
    xmlns:html="http://www.w3.org/1999/xhtml">
    <!--  ↑           ↑
      prefijo    URI (identificador único, no necesita existir como URL)  -->

    <!-- "tabla" de muebles (prefijo "mueble:") -->
    <mueble:tabla>
        <mueble:material>madera</mueble:material>
        <mueble:patas>4</mueble:patas>
    </mueble:tabla>

    <!-- "tabla" HTML (prefijo "html:") -->
    <html:table>
        <html:tr>
            <html:td>Dato 1</html:td>
            <html:td>Dato 2</html:td>
        </html:tr>
    </html:table>

    <!-- ✅ Sin conflictos: mueble:tabla ≠ html:table -->
</documento>

    ❌ Errores Comunes en XML

    • Olvidar cerrar etiquetas → XML es estricto, TODAS las etiquetas deben cerrarse
    • Mayúsculas/minúsculas inconsistentes → <Nombre> ≠ <nombre> (case-sensitive)
    • Más de un elemento raíz → Solo puede haber UN elemento raíz
    • Atributos sin comillas → nombre=valor → debe ser nombre="valor"
    • Atributos duplicados → <libro id="1" id="2"> → No permitido
    • Caracteres especiales sin escapar → Usa &lt; &gt; &amp; &apos; &quot;
    • Etiquetas cruzadas → <a><b></a></b> → Anidación incorrecta
    • Espacios en nombres de elementos → <mi elemento> → Usa <mi_elemento>
    • Olvidar encoding UTF-8 → Caracteres especiales (ñ, á, etc.) pueden fallar
    • Confundir XML con HTML → En XML <br> debe ser <br/>

    💡 Consejos Pro XML

    • Usa UTF-8 siempre: <?xml version="1.0" encoding="UTF-8"?>
    • Nomenclatura consistente: Elige snake_case o camelCase y úsalo siempre
    • Valida tus XMLs: Usa herramientas online o editores con validación
    • Atributos para metadatos: id, tipo, categoría → elementos para datos principales
    • Indentación clara: Facilita la lectura (2 o 4 espacios, no tabs)
    • Comentarios descriptivos: Documenta secciones complejas
    • CDATA para código: <![CDATA[...]]> evita escapar < > &
    • Namespaces en proyectos grandes: Evitan conflictos al combinar vocabularios
    • Nombres auto-descriptivos: <fecha_nacimiento> mejor que <fecha1>
    • Estructura antes de contenido: Diseña el esquema antes de escribir el XML
    • Herramientas útiles: XMLStarlet, xmllint para validación en terminal
    • Pretty print: Formatea el XML con herramientas antes de entregar

    DTD - Document Type Definition

    Introducción a DTD

    DTD (Document Type Definition) es el sistema clásico para definir la estructura y reglas de validación de documentos XML. Funciona como un contrato que especifica qué elementos, atributos y estructura debe tener un XML válido.

    🎯 ¿Para qué sirve DTD?

    • Validación: Verificar que un XML cumple con las reglas definidas
    • Documentación: Definir la estructura esperada del documento
    • Estandarización: Asegurar que todos los documentos sigan el mismo formato
    • Interoperabilidad: Facilitar el intercambio de datos entre sistemas
    • Valores por defecto: Definir valores predeterminados para atributos
    Ejemplo Básico de DTD
    <?xml version="1.0" encoding="UTF-8"?>

<!-- DTD interno en el mismo archivo XML -->
<!DOCTYPE biblioteca [
    <!-- Definición de elementos -->
    <!ELEMENT biblioteca (libro+)>
    <!ELEMENT libro (titulo, autor, año, precio)>
    <!ELEMENT titulo (#PCDATA)>
    <!ELEMENT autor (#PCDATA)>
    <!ELEMENT año (#PCDATA)>
    <!ELEMENT precio (#PCDATA)>

    <!-- Definición de atributos -->
    <!ATTLIST libro
        id ID #REQUIRED
        categoria CDATA #IMPLIED
    >
]>

<biblioteca>
    <libro id="L001" categoria="Ficción">
        <titulo>Don Quijote de la Mancha</titulo>
        <autor>Miguel de Cervantes</autor>
        <año>1605</año>
        <precio>25.99</precio>
    </libro>

    <libro id="L002" categoria="Programación">
        <titulo>Clean Code</titulo>
        <autor>Robert C. Martin</autor>
        <año>2008</año>
        <precio>39.99</precio>
    </libro>
</biblioteca>

    📝 Tipos de DTD

    DTD Interno: Se define dentro del mismo archivo XML usando <!DOCTYPE nombre [...]>

    DTD Externo: Se almacena en un archivo .dtd separado y se referencia desde el XML

    Sintaxis de Elementos DTD

    Declaración de Elementos
    <!-- SINTAXIS BÁSICA -->
<!ELEMENT nombre_elemento contenido>


<!-- 1. ELEMENTO CON TEXTO (#PCDATA = Parsed Character Data) -->
<!ELEMENT titulo (#PCDATA)>
<!-- Ejemplo: <titulo>Mi libro favorito</titulo> -->


<!-- 2. ELEMENTO VACÍO (EMPTY) -->
<!ELEMENT br EMPTY>
<!-- Ejemplo: <br/> -->


<!-- 3. ELEMENTO CON CUALQUIER CONTENIDO (ANY) -->
<!ELEMENT contenido ANY>
<!-- Puede contener texto, elementos, o mezcla (no recomendado) -->


<!-- 4. ELEMENTO CON SECUENCIA DE HIJOS -->
<!ELEMENT libro (titulo, autor, año)>
<!--           ↑
         Deben aparecer EN ESTE ORDEN exacto  -->
<!-- Ejemplo:
<libro>
    <titulo>...</titulo>
    <autor>...</autor>
    <año>...</año>
</libro> -->


<!-- 5. ELEMENTO CON HIJOS OPCIONALES (elección) -->
<!ELEMENT contacto (email | telefono)>
<!--                ↑
         Uno u otro, pero NO ambos  -->
<!-- Ejemplo válido: <contacto><email>...</email></contacto> -->
<!-- Ejemplo válido: <contacto><telefono>...</telefono></contacto> -->
    Indicadores de Ocurrencia
    <!-- INDICADORES DE OCURRENCIA -->

<!-- (+) = UNO O MÁS (mínimo 1) -->
<!ELEMENT biblioteca (libro+)>
<!-- La biblioteca DEBE tener AL MENOS un libro -->


<!-- (*) = CERO O MÁS -->
<!ELEMENT comentarios (comentario*)>
<!-- Puede tener 0, 1, 2, ... infinitos comentarios -->


<!-- (?) = CERO O UNO (opcional) -->
<!ELEMENT libro (titulo, autor, subtitulo?, año)>
<!--                              ↑
         Subtítulo es OPCIONAL, puede existir o no  -->


<!-- SIN INDICADOR = EXACTAMENTE UNO (obligatorio) -->
<!ELEMENT libro (titulo, autor)>
<!-- Título y autor son OBLIGATORIOS, aparecen 1 vez -->
    Ejemplos Combinados Complejos
    <!-- COMBINACIONES COMPLEJAS -->

<!-- Ejemplo 1: Secuencia con opcionales -->
<!ELEMENT articulo (titulo, subtitulo?, autor+, fecha, (resumen | extracto), contenido, referencias*)>
<!--
    - titulo: obligatorio (1)
    - subtitulo: opcional (0 o 1)
    - autor: al menos 1, puede haber varios
    - fecha: obligatorio (1)
    - resumen o extracto: uno u otro, obligatorio
    - contenido: obligatorio (1)
    - referencias: 0 o más
-->


<!-- Ejemplo 2: Grupo de elección -->
<!ELEMENT persona (nombre, (telefono | email | (telefono, email)), direccion?)>
<!--
    - nombre: obligatorio
    - telefono o email o ambos: al menos uno obligatorio
    - direccion: opcional
-->


<!-- Ejemplo 3: Contenido mixto (texto + elementos) -->
<!ELEMENT parrafo (#PCDATA | negrita | cursiva)*>
<!-- Puede contener texto y etiquetas de formato mezcladas -->
<!-- Ejemplo: <parrafo>Este es <negrita>importante</negrita> y esto <cursiva>cursiva</cursiva></parrafo> -->


<!-- Ejemplo 4: Estructura recursiva -->
<!ELEMENT lista (item+)>
<!ELEMENT item (#PCDATA | lista)*>
<!-- Permite listas anidadas infinitamente -->

    Sintaxis de Atributos DTD

    Declaración de Atributos
    <!-- SINTAXIS BÁSICA -->
<!ATTLIST nombre_elemento
    nombre_atributo tipo_dato modo
    nombre_atributo2 tipo_dato modo
>


<!-- TIPOS DE DATOS MÁS COMUNES -->

<!-- 1. CDATA = Cualquier texto -->
<!ATTLIST libro
    titulo CDATA #REQUIRED
>


<!-- 2. ID = Identificador único en el documento -->
<!ATTLIST libro
    id ID #REQUIRED
>
<!-- Ejemplo: <libro id="L001"> -->
<!-- Cada ID debe ser ÚNICO en todo el documento -->


<!-- 3. IDREF = Referencia a un ID existente -->
<!ATTLIST prestamo
    libro_ref IDREF #REQUIRED
>
<!-- Ejemplo: <prestamo libro_ref="L001"> -->
<!-- Debe referenciar a un id existente -->


<!-- 4. IDREFS = Lista de referencias a IDs -->
<!ATTLIST pedido
    libros_ref IDREFS #REQUIRED
>
<!-- Ejemplo: <pedido libros_ref="L001 L003 L007"> -->


<!-- 5. NMTOKEN = Token sin espacios -->
<!ATTLIST producto
    codigo NMTOKEN #REQUIRED
>
<!-- Ejemplo: <producto codigo="ABC-123"> -->


<!-- 6. ENUMERACIÓN = Lista cerrada de valores -->
<!ATTLIST libro
    categoria (Ficcion|Tecnologia|Historia|Ciencia) #REQUIRED
>
<!-- Solo puede ser uno de esos valores exactos -->
    Modos de Atributos
    <!-- MODOS DE ATRIBUTOS -->

<!-- 1. #REQUIRED = Obligatorio -->
<!ATTLIST libro
    id ID #REQUIRED
>
<!-- El atributo DEBE aparecer siempre -->


<!-- 2. #IMPLIED = Opcional -->
<!ATTLIST libro
    editorial CDATA #IMPLIED
>
<!-- El atributo puede aparecer o no -->


<!-- 3. #FIXED = Valor fijo -->
<!ATTLIST libro
    version CDATA #FIXED "1.0"
>
<!-- Si aparece, DEBE tener este valor exacto -->
<!-- Ejemplo: <libro version="1.0"> -->


<!-- 4. VALOR POR DEFECTO -->
<!ATTLIST libro
    moneda CDATA "EUR"
>
<!-- Si no se especifica, se usa "EUR" -->
<!-- Ejemplo: <libro> (usará moneda="EUR" automáticamente) -->

    ❌ Limitaciones de DTD

    • No soporta tipos de datos: Todo es texto (no hay integer, date, boolean, etc.)
    • No soporta namespaces nativamente: Dificulta trabajar con múltiples vocabularios
    • Sintaxis no-XML: DTD no usa sintaxis XML, es un lenguaje propio
    • Limitada expresividad: No puede definir restricciones complejas (rangos, patrones, etc.)
    • No soporta herencia: No hay mecanismos de extensión o reutilización avanzada
    • Difícil de procesar: Requiere un parser específico, no puede usar un parser XML estándar

    Solución: Para proyectos modernos, se recomienda usar XML Schema (XSD) que resuelve todas estas limitaciones.

    💡 Consejos Pro DTD

    • Usa DTD para proyectos sencillos: Si solo necesitas validación básica de estructura
    • Prefiere XSD para proyectos complejos: Si necesitas validación de tipos de datos
    • Usa entidades para texto repetido: Evita duplicación de información
    • DTD externo para reutilización: Un DTD puede servir para múltiples XMLs
    • Documenta tus DTDs: Usa comentarios para explicar la estructura
    • Herramientas de validación: xmllint (Linux/Mac), editores como VS Code con extensiones

    XSD - XML Schema Definition

    Introducción a XML Schema (XSD)

    XML Schema (XSD) es el estándar moderno del W3C para definir la estructura, contenido y tipos de datos de documentos XML. Es el sucesor de DTD y ofrece capacidades mucho más potentes y flexibles.

    🚀 Ventajas de XSD sobre DTD

    • Tipos de datos ricos: string, integer, date, boolean, decimal, etc.
    • Sintaxis XML: XSD está escrito en XML, puede procesarse con parsers XML estándar
    • Soporte de namespaces: Integración nativa con espacios de nombres
    • Restricciones avanzadas: Rangos, patrones (regex), longitudes, enumeraciones
    • Herencia y extensión: Reutilización mediante extensión y restricción de tipos
    • Documentación integrada: Elemento <xs:annotation> para documentar
    • Tipos complejos: Definición de estructuras complejas reutilizables
    • Mejor validación: Validación de tipos de datos y restricciones precisas
    Comparación DTD vs XSD
    <!-- DTD: Sintaxis propia, limitada -->
<!ELEMENT libro (titulo, autor, año, precio)>
<!ELEMENT titulo (#PCDATA)>
<!ELEMENT año (#PCDATA)>  <!-- No puede validar que sea un número -->


<!-- XSD: Sintaxis XML, tipos de datos -->
<xs:element name="libro">
    <xs:complexType>
        <xs:sequence>
            <xs:element name="titulo" type="xs:string"/>
            <xs:element name="autor" type="xs:string"/>
            <xs:element name="año" type="xs:integer"/>  <!-- Valida que sea entero -->
            <xs:element name="precio" type="xs:decimal"/>  <!-- Valida que sea decimal -->
        </xs:sequence>
    </xs:complexType>
</xs:element>

    Restricciones (Facets) en XSD

    Las restricciones permiten definir reglas específicas sobre los valores que puede tomar un elemento o atributo.

    Restricciones de Longitud
    <!-- LONGITUD EXACTA -->
<xs:simpleType name="CodigoPostal">
    <xs:restriction base="xs:string">
        <xs:length value="5"/>
    </xs:restriction>
</xs:simpleType>
<!-- Ejemplo válido: 28013 -->
<!-- Ejemplo inválido: 280 (menos de 5 caracteres) -->


<!-- LONGITUD MÍNIMA -->
<xs:simpleType name="Contraseña">
    <xs:restriction base="xs:string">
        <xs:minLength value="8"/>
    </xs:restriction>
</xs:simpleType>
<!-- Mínimo 8 caracteres -->


<!-- LONGITUD MÁXIMA -->
<xs:simpleType name="Resumen">
    <xs:restriction base="xs:string">
        <xs:maxLength value="200"/>
    </xs:restriction>
</xs:simpleType>
<!-- Máximo 200 caracteres -->


<!-- COMBINANDO MÍNIMO Y MÁXIMO -->
<xs:simpleType name="NombreUsuario">
    <xs:restriction base="xs:string">
        <xs:minLength value="3"/>
        <xs:maxLength value="20"/>
    </xs:restriction>
</xs:simpleType>
<!-- Entre 3 y 20 caracteres -->
    Restricciones de Rango (Numéricos)
    <!-- VALOR MÍNIMO (inclusivo) -->
<xs:simpleType name="Edad">
    <xs:restriction base="xs:integer">
        <xs:minInclusive value="0"/>
        <xs:maxInclusive value="120"/>
    </xs:restriction>
</xs:simpleType>
<!-- Entre 0 y 120 (ambos incluidos) -->


<!-- VALOR MÍNIMO (exclusivo) -->
<xs:simpleType name="NotaExamen">
    <xs:restriction base="xs:decimal">
        <xs:minExclusive value="0"/>
        <xs:maxInclusive value="10"/>
    </xs:restriction>
</xs:simpleType>
<!-- Mayor que 0 (exclusivo) hasta 10 (inclusivo) -->
<!-- Válido: 0.01, 5.5, 10.0 -->
<!-- Inválido: 0, -1, 10.5 -->


<!-- PRECIO CON DECIMALES LIMITADOS -->
<xs:simpleType name="Precio">
    <xs:restriction base="xs:decimal">
        <xs:minInclusive value="0.01"/>
        <xs:maxInclusive value="99999.99"/>
        <xs:fractionDigits value="2"/>  <!-- Máximo 2 decimales -->
    </xs:restriction>
</xs:simpleType>
<!-- Válido: 19.99, 100.00 -->
<!-- Inválido: 19.999 (3 decimales) -->

<xs:simpleType name="CodigoProducto">
    <xs:restriction base="xs:integer">
        <xs:totalDigits value="8"/>
    </xs:restriction>
</xs:simpleType>
<!-- Máximo 8 dígitos: 12345678 -->

    Secuencias (xs:sequence)
    <!-- SECUENCIA: elementos deben aparecer EN ORDEN -->
<xs:element name="libro">

        <xs:sequence>
            <xs:element name="titulo" type="xs:string"/>
            <xs:element name="autor" type="xs:string"/>
            <xs:element name="año" type="xs:integer"/>
            <xs:element name="precio" type="xs:decimal"/>
        </xs:sequence>
    </xs:complexType>
</xs:element>

<xs:element name="contacto">

            <xs:choice>

                <xs:element name="telefono" type="xs:string"/>
            </xs:choice>
        </xs:sequence>
    </xs:complexType>

            <xs:element name="descripcion" type="xs:string" minOccurs="0"/>

                    <xs:sequence>
                        <xs:element name="titulo" type="xs:string"/>

                </xs:complexType>
            </xs:element>

            <xs:element name="comentario" type="xs:string"
                        minOccurs="0" maxOccurs="unbounded"/>
                <!--                        ↑
                     unbounded = sin límite (infinito)  -->
        </xs:sequence>
    </xs:complexType>

    JSON - JavaScript Object Notation

    Introducción a JSON

    JSON (JavaScript Object Notation) es un formato ligero de intercambio de datos basado en texto, fácil de leer y escribir para humanos, y fácil de parsear y generar para máquinas. Es el estándar de facto para APIs REST y comunicación web moderna.

    🎯 ¿Por qué usar JSON?

    • Ligero y compacto: Menos verbose que XML, ideal para APIs y transferencia de datos
    • Fácil de leer: Sintaxis simple y clara, muy intuitiva
    • Universal: Soportado nativamente por JavaScript y prácticamente todos los lenguajes
    • Rápido de parsear: Procesamiento eficiente en navegadores y servidores
    • Tipado dinámico: Strings, números, booleanos, arrays, objetos, null
    • Perfecto para APIs REST: Formato estándar para comunicación HTTP

    ❌ Errores Comunes en JSON

    • Comillas simples: JSON solo acepta comillas dobles ", no simples '
    • Comas finales: {"a": 1,} → No permitidas en JSON
    • Claves sin comillas: {nombre: "Ana"} → Debe ser {"nombre": "Ana"}
    • Comentarios: JSON estándar no permite comentarios // ni /* */
    • Valores undefined: JSON no soporta undefined, usa null
    • Funciones: JSON no puede almacenar funciones, solo datos
    • Fechas: No hay tipo Date, usa strings ISO: "2025-01-18T10:00:00Z"
    • NaN e Infinity: No soportados, se convierten a null

    💡 Consejos Pro JSON

    • Valida tu JSON: Usa herramientas como JSONLint para verificar sintaxis
    • Formatea para legibilidad: Usa JSON.stringify(obj, null, 2) para pretty print
    • Nomenclatura camelCase: Convención estándar en JSON: firstName, no first_name
    • Fechas en formato ISO 8601: "2025-01-18T10:00:00Z" para compatibilidad universal
    • Usa arrays para listas: Más eficiente y fácil de iterar que objetos con claves numéricas
    • Try-catch al parsear: Siempre maneja errores: try { JSON.parse(str) } catch(e) { ... }
    • Evita anidación excesiva: Más de 3-4 niveles dificulta lectura y mantenimiento
    • JSON Schema: Para validación avanzada de estructura JSON

    XSLT - XSL Transformations

    Introducción a XSLT

    XSLT (eXtensible Stylesheet Language Transformations) es un lenguaje para transformar documentos XML en otros formatos: HTML, texto, otro XML, PDF, etc. Es como CSS para XML, pero mucho más potente: puede reordenar, filtrar, calcular y generar contenido dinámico.

    🎯 ¿Para qué sirve XSLT?

    • Transformar XML a HTML: Mostrar datos XML en páginas web
    • Filtrar datos: Seleccionar solo ciertos elementos de un XML grande
    • Ordenar datos: Reorganizar elementos por criterios específicos
    • Calcular valores: Sumar, contar, promediar datos
    • Generar reportes: Crear documentos formateados a partir de datos XML
    • Migración de formatos: Convertir entre diferentes esquemas XML

                    <div class="productos">
                        <!-- Ordenar por precio descendente -->
                        <xsl:for-each select="tienda/productos/producto">
                            <xsl:sort select="precio" data-type="number" order="descending"/>

                            <!-- Aplicar clase "destacado" si el atributo lo indica -->
                            <div>
                                <xsl:attribute name="class">
                                    producto
                                    <xsl:if test="@destacado = 'si'"> destacado</xsl:if>
                                </xsl:attribute>

                                <!-- Mostrar badge si es destacado -->
                                <xsl:if test="@destacado = 'si'">
                                    <span class="badge">DESTACADO</span>
                                </xsl:if>

                                <div class="nombre"><xsl:value-of select="nombre"/></div>
                                <div class="categoria"><xsl:value-of select="categoria"/></div>
                                <div class="precio"><xsl:value-of select="precio"/> €</div>

                                <div class="stock">
                                    Stock:
                                    <xsl:choose>
                                        <xsl:when test="stock &gt; 20">
                                            <span style="color: green;">✓ Disponible (<xsl:value-of select="stock"/>)</span>
                                        </xsl:when>
                                        <xsl:when test="stock &gt; 0">
                                            <span style="color: orange;">⚠ Pocas unidades (<xsl:value-of select="stock"/>)</span>
                                        </xsl:when>
                                        <xsl:otherwise>
                                            <span style="color: red;">✗ Agotado</span>
                                        </xsl:otherwise>
                                    </xsl:choose>
                                </div>

                                <div class="valoracion">
                                    ★ <xsl:value-of select="valoracion"/> / 5.0
                                </div>
                            </div>
                        </xsl:for-each>
                    </div>
                </div>
            </body>
        </html>
    </xsl:template>

</xsl:stylesheet>

    💡 Consejos Pro XSLT

    • XPath es fundamental: Domina selectores XPath para XSLT eficiente
    • Templates vs for-each: Usa templates para reutilización, for-each para iteraciones simples
    • Ordena con xsl:sort: Dentro de for-each para ordenar resultados
    • Filtra con predicados: libro[precio > 30] selecciona solo libros caros
    • Funciones útiles: count(), sum(), concat(), substring()
    • Escapa caracteres: &lt; para <, &gt; para >
    • Herramientas de prueba: Navegadores modernos, editores XML con vista previa XSLT
