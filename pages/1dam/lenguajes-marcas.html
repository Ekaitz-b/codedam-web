    HTML5 - Estructura Web

    HTML (HyperText Markup Language) es el lenguaje estándar para crear páginas web. Define la estructura y el contenido.

    Estructura Básica

    <!DOCTYPE html> <html lang="es"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Mi Primera Página</title> <link rel="stylesheet" href="styles.css"> </head> <body> <header> <h1>Bienvenido a CodeDAM</h1> <nav> <a href="#inicio">Inicio</a> <a href="#cursos">Cursos</a> </nav> </header> <main> <section> <h2>Contenido Principal</h2> <p>Este es un párrafo de ejemplo.</p> </section> </main> <footer> <p>© 2025 CodeDAM</p> </footer> <script src="script.js"></script> </body> </html>

    Etiquetas Semánticas HTML5

    • <header> - Cabecera de página o sección
    • <nav> - Navegación
    • <main> - Contenido principal
    • <section> - Sección temática
    • <article> - Contenido independiente
    • <aside> - Contenido relacionado lateral
    • <footer> - Pie de página

    Formularios HTML

    <form action="/enviar" method="POST"> <label for="nombre">Nombre:</label> <input type="text" id="nombre" name="nombre" required> <label for="email">Email:</label> <input type="email" id="email" name="email" required> <label for="edad">Edad:</label> <input type="number" id="edad" name="edad" min="18" max="100"> <label for="curso">Curso:</label> <select id="curso" name="curso"> <option value="1dam">1º DAM</option> <option value="2dam">2º DAM</option> </select> <textarea name="comentarios" rows="4"></textarea> <button type="submit">Enviar</button> </form>

    CSS3 - Estilos y Diseño

    CSS (Cascading Style Sheets) controla la presentación visual de las páginas web.

    Selectores CSS

    /* Selector de elemento */ p { color: #333; font-size: 16px; } /* Selector de clase */ .destacado { background-color: yellow; font-weight: bold; } /* Selector de ID */ #header { background-color: #667eea; padding: 20px; } /* Selectores combinados */ .container p { margin-bottom: 15px; } header nav a { text-decoration: none; color: white; } /* Pseudoclases */ a:hover { color: #764ba2; } button:active { transform: scale(0.95); } li:nth-child(odd) { background-color: #f5f5f5; }

    Flexbox - Diseño Flexible

    .container { display: flex; justify-content: space-between; align-items: center; gap: 20px; } .sidebar { flex: 0 0 300px; /* No crece, no encoge, 300px base */ } .main-content { flex: 1; /* Crece para llenar espacio disponible */ } /* Responsive con flex-wrap */ .cards { display: flex; flex-wrap: wrap; gap: 20px; } .card { flex: 0 0 calc(33.333% - 20px); }

    Grid - Diseño en Cuadrícula

    .grid-container { display: grid; grid-template-columns: repeat(3, 1fr); grid-gap: 20px; } /* Grid con áreas nombradas */ .layout { display: grid; grid-template-areas: "header header header" "sidebar main main" "footer footer footer"; grid-template-columns: 250px 1fr 1fr; } .header { grid-area: header; } .sidebar { grid-area: sidebar; } .main { grid-area: main; } .footer { grid-area: footer; }

    Responsive Design

    /* Mobile First */ .container { width: 100%; padding: 10px; } /* Tablet */ @media (min-width: 768px) { .container { max-width: 720px; margin: 0 auto; } } /* Desktop */ @media (min-width: 1024px) { .container { max-width: 960px; } .grid { grid-template-columns: repeat(3, 1fr); } }

    XML - Lenguaje de Marcado Extensible

    XML se usa para almacenar y transportar datos de forma estructurada y legible.

    Estructura XML

    <?xml version="1.0" encoding="UTF-8"?> <biblioteca> <libro isbn="978-0134685991"> <titulo>Effective Java</titulo> <autor>Joshua Bloch</autor> <editorial>Addison-Wesley</editorial> <año>2018</año> <precio moneda="EUR">45.50</precio> </libro> <libro isbn="978-0596009205"> <titulo>Head First Java</titulo> <autor>Kathy Sierra</autor> <editorial>O'Reilly</editorial> <año>2005</año> <precio moneda="EUR">39.99</precio> </libro> </biblioteca>

    DTD - Definición de Tipo de Documento

    <!DOCTYPE biblioteca [ <!ELEMENT biblioteca (libro+)> <!ELEMENT libro (titulo, autor, editorial, año, precio)> <!ATTLIST libro isbn CDATA #REQUIRED> <!ELEMENT titulo (#PCDATA)> <!ELEMENT autor (#PCDATA)> <!ELEMENT editorial (#PCDATA)> <!ELEMENT año (#PCDATA)> <!ELEMENT precio (#PCDATA)> <!ATTLIST precio moneda CDATA #IMPLIED> ]>

    JSON - Intercambio de Datos

    JSON (JavaScript Object Notation) es el formato más popular para APIs y almacenamiento de datos.

    Sintaxis JSON

    { "nombre": "Ana García", "edad": 20, "estudiante": true, "curso": "2º DAM", "asignaturas": [ "Acceso a Datos", "Desarrollo de Interfaces", "Programación de Servicios" ], "direccion": { "calle": "Calle Principal", "numero": 123, "ciudad": "Madrid", "codigoPostal": "28001" }, "notas": { "accesoADatos": 8.5, "desarrolloInterfaces": 9.0, "programacionServicios": 8.0 } }

    Comparación XML vs JSON

    // XML <persona> <nombre>Ana</nombre> <edad>20</edad> </persona> // JSON (más compacto) { "nombre": "Ana", "edad": 20 }

    Cuándo usar cada uno: XML para documentos complejos y configuración. JSON para APIs y datos simples.

    Mejores Prácticas Web

    • Usa HTML semántico para mejor SEO y accesibilidad
    • Separa contenido (HTML), presentación (CSS) y comportamiento (JS)
    • Diseña primero para móvil (Mobile First)
    • Optimiza imágenes y recursos
    • Valida tu código HTML y CSS
    • Usa nombres descriptivos para clases e IDs
    • Indenta correctamente tu código
    • Comenta secciones complejas

    Errores comunes: Olvidar etiquetas de cierre, usar IDs duplicados, no cerrar etiquetas en XML.
