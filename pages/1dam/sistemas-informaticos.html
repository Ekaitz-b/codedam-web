    Arquitectura de Computadores

    Componentes Principales del Hardware

    CPU (Unidad Central de Procesamiento)

    El procesador es el cerebro del ordenador. Ejecuta las instrucciones de los programas realizando operaciones aritméticas, lógicas y de control.

    • Unidad de Control (CU): Interpreta instrucciones y coordina la ejecución
    • Unidad Aritmético-Lógica (ALU): Realiza cálculos matemáticos y comparaciones
    • Registros: Memoria ultrarrápida dentro de la CPU para datos temporales
    • Caché: Memoria intermedia (L1, L2, L3) que acelera el acceso a datos frecuentes
    # Especificaciones típicas de CPUs actuales Intel Core i5-13400: - 10 núcleos (6 Performance + 4 Efficiency) - 16 hilos de ejecución - Frecuencia base: 2.5 GHz, Turbo: 4.6 GHz - Caché L3: 20 MB - TDP: 65W AMD Ryzen 5 7600: - 6 núcleos, 12 hilos - Frecuencia base: 3.8 GHz, Turbo: 5.1 GHz - Caché L3: 32 MB - TDP: 65W

    RAM (Memoria de Acceso Aleatorio)

    Almacenamiento temporal volátil donde se cargan los programas y datos en uso. Características principales:

    • Volátil: Se borra al apagar el equipo
    • Acceso aleatorio: Cualquier posición accesible en tiempo constante
    • Tipos actuales: DDR4 (3200 MHz típico) y DDR5 (4800-6400 MHz)
    • Latencia: CL16-CL18 para DDR4, CL30-CL40 para DDR5
    # Configuraciones típicas de RAM Uso básico/ofimática: 8 GB DDR4-3200 Desarrollo/Gaming: 16 GB DDR4-3600 (2x8 GB dual channel) Profesional/Workstation: 32-64 GB DDR5-5600 Servidores: 128+ GB ECC DDR5

    Almacenamiento

    • SSD NVMe: Hasta 7000 MB/s lectura, conexión PCIe directa. Ideal para SO y aplicaciones
    • SSD SATA: Hasta 550 MB/s, más económicos, formato 2.5"
    • HDD: 100-200 MB/s, alta capacidad (hasta 20TB), ideal para almacenamiento masivo

    Buses y Comunicación Interna

    Los buses son canales de comunicación que conectan los componentes del sistema:

    • Bus de datos: Transporta datos entre componentes (64 bits en sistemas modernos)
    • Bus de direcciones: Indica la ubicación de memoria a acceder
    • Bus de control: Transmite señales de sincronización y control

    Interfaces de Conexión Modernas

    PCIe 4.0 x16: ~32 GB/s (tarjetas gráficas) PCIe 4.0 x4: ~8 GB/s (SSDs NVMe) PCIe 5.0 x4: ~16 GB/s (SSDs de última generación) SATA III: ~600 MB/s (discos SATA) USB 3.2 Gen2: ~1.25 GB/s (periféricos) USB4/TB4: ~5 GB/s (docks, almacenamiento externo)

    Periféricos de Entrada/Salida

    • Entrada: Teclado, ratón, escáner, micrófono, webcam, lectores de tarjetas
    • Salida: Monitor, impresora, altavoces, auriculares
    • E/S: Pantallas táctiles, dispositivos de almacenamiento externo, modems

    Sistemas Operativos Fundamentales

    ¿Qué es un Sistema Operativo?

    El Sistema Operativo (SO) es el software fundamental que gestiona el hardware y proporciona servicios a las aplicaciones. Actúa como intermediario entre el usuario y el hardware.

    Funciones Principales del SO

    • Gestión de procesos: Crear, planificar, ejecutar y terminar procesos
    • Gestión de memoria: Asignar y liberar memoria RAM a los procesos
    • Gestión de archivos: Organizar, almacenar y recuperar datos en disco
    • Gestión de E/S: Controlar dispositivos de entrada/salida
    • Seguridad: Control de acceso, permisos, autenticación
    • Interfaz de usuario: CLI (línea de comandos) o GUI (gráfica)

    Gestión de Procesos

    Un proceso es un programa en ejecución con su propio espacio de memoria, contexto y recursos.

    # Estados de un proceso NUEVO: Proceso recién creado LISTO: Esperando CPU para ejecutarse EJECUTANDO: Usando la CPU activamente BLOQUEADO: Esperando un evento (E/S, recurso) TERMINADO: Ejecución finalizada # Algoritmos de planificación comunes FCFS (First Come First Served): Cola simple, no prioritario Round Robin: Turnos con quantum de tiempo fijo Prioridades: Procesos con mayor prioridad primero Multinivel: Varias colas con diferentes políticas

    Gestión de Memoria

    • Memoria virtual: Usa disco como extensión de RAM (swap/paginación)
    • Paginación: Divide memoria en páginas de tamaño fijo (típico 4 KB)
    • Segmentación: Divide memoria en segmentos lógicos de tamaño variable
    • Protección: Aísla procesos para evitar accesos no autorizados

    Gestión del Sistema de Archivos

    # Sistemas de archivos comunes Windows: NTFS: Sistema principal de Windows (permisos, journaling, compresión) FAT32: Compatible pero limitado (máx 4GB por archivo) exFAT: Para unidades externas, sin límite de 4GB Linux: ext4: Sistema estándar de Linux (journaling, extensiones) XFS: Alto rendimiento para archivos grandes Btrfs: Moderno con snapshots y RAID integrado macOS: APFS: Sistema moderno de Apple (encriptación, snapshots)

    Virtualización Básica

    La virtualización permite ejecutar múltiples sistemas operativos en un mismo hardware físico.

    • Tipo 1 (Bare-metal): Hipervisor sobre hardware (VMware ESXi, Hyper-V Server)
    • Tipo 2 (Hosted): Hipervisor sobre SO (VirtualBox, VMware Workstation)
    • Contenedores: Virtualización a nivel de SO (Docker, Podman)

    Permisos en Linux

    Linux usa un sistema de permisos basado en usuario, grupo y otros, con permisos de lectura (r), escritura (w) y ejecución (x).

    # Interpretar permisos (ls -l) -rwxr-xr-- 1 usuario grupo 4096 mar 15 10:30 script.sh - = tipo (- archivo, d directorio, l enlace) rwx = propietario (read, write, execute) r-x = grupo (read, execute) r-- = otros (read only) # Notación octal r = 4, w = 2, x = 1 755 = rwxr-xr-x (propietario todo, grupo y otros lectura+ejecución) 644 = rw-r--r-- (propietario lectura+escritura, resto solo lectura) 700 = rwx------ (solo propietario tiene acceso) 777 = rwxrwxrwx (todos los permisos para todos - EVITAR)

    Comandos chmod y chown

    # Cambiar permisos con notación octal chmod 755 script.sh # Ejecutable para todos chmod 644 documento.txt # Lectura para todos, escritura propietario chmod 600 clave_privada # Solo propietario puede leer/escribir # Cambiar permisos con notación simbólica chmod +x script.sh # Añadir ejecución para todos chmod u+w archivo.txt # Añadir escritura al usuario (user) chmod g-w archivo.txt # Quitar escritura al grupo chmod o-rwx secreto.txt # Quitar todos los permisos a otros chmod a+r publico.txt # Lectura para todos (all) # Recursivo en directorios chmod -R 755 /var/www/html/ # Aplicar a todo el contenido # Cambiar propietario chown usuario archivo.txt # Cambiar usuario propietario chown usuario:grupo archivo.txt # Cambiar usuario y grupo chown -R www-data:www-data /var/www # Recursivo # Cambiar solo grupo chgrp developers proyecto/ # Ejemplo práctico: servidor web sudo chown -R www-data:www-data /var/www/html sudo chmod -R 755 /var/www/html sudo chmod -R 644 /var/www/html/*.html

    Redes Informáticas

    Modelo OSI - Las 7 Capas

    El modelo OSI (Open Systems Interconnection) es un modelo de referencia para entender cómo se comunican los sistemas en red.

    Capa Nombre Función Protocolos/Dispositivos
    7 Aplicación Interfaz con aplicaciones de usuario HTTP, HTTPS, FTP, SMTP, DNS, SSH
    6 Presentación Formato de datos, encriptación, compresión SSL/TLS, JPEG, ASCII, MPEG
    5 Sesión Establecer, gestionar y terminar sesiones NetBIOS, RPC, SQL sessions
    4 Transporte Transmisión fiable extremo a extremo TCP, UDP. Puertos
    3 Red Direccionamiento y enrutamiento IP, ICMP, Routers
    2 Enlace de Datos Transmisión entre nodos adyacentes Ethernet, WiFi, MAC, Switches
    1 Física Transmisión de bits por medio físico Cables, conectores, hubs, señales

    Regla mnemotécnica: "Please Do Not Throw Sausage Pizza Away" (Physical, Data Link, Network, Transport, Session, Presentation, Application)

    Modelo TCP/IP en Profundidad

    El modelo TCP/IP es el modelo práctico usado en Internet, con 4 capas que engloban las funciones del modelo OSI.

    Comparación OSI vs TCP/IP: OSI (7 capas) TCP/IP (4 capas) ───────────── ──────────────── 7. Aplicación ┐ 6. Presentación ├─→ 4. Aplicación (HTTP, FTP, DNS, SMTP) 5. Sesión ┘ 4. Transporte ──→ 3. Transporte (TCP, UDP) 3. Red ──→ 2. Internet (IP, ICMP, ARP) 2. Enlace ┐ 1. Física ┴─→ 1. Acceso a Red (Ethernet, WiFi)

    TCP vs UDP

    TCP (Transmission Control Protocol): - Orientado a conexión (handshake de 3 vías) - Garantiza entrega ordenada y sin errores - Control de flujo y congestión - Más lento pero fiable - Usos: HTTP, HTTPS, FTP, SMTP, SSH UDP (User Datagram Protocol): - Sin conexión (connectionless) - No garantiza entrega ni orden - Sin control de flujo - Más rápido pero no fiable - Usos: DNS, DHCP, streaming, videollamadas, gaming # Puertos comunes TCP 20-21: FTP (datos y control) TCP 22: SSH TCP 23: Telnet TCP 25: SMTP TCP 53: DNS (también UDP) TCP 80: HTTP TCP 443: HTTPS TCP 3306: MySQL TCP 5432: PostgreSQL UDP 53: DNS UDP 67-68: DHCP UDP 123: NTP

    Direccionamiento IP

    IPv4

    # Formato: 4 octetos de 8 bits (32 bits total) # Rango por octeto: 0-255 192.168.1.100 │ │ │ │ │ │ │ └── Host │ │ └───── Subred │ └───────── Red └───────────── Red # Clases de direcciones (histórico) Clase A: 1.0.0.0 - 126.255.255.255 (/8, 16 millones hosts) Clase B: 128.0.0.0 - 191.255.255.255 (/16, 65,534 hosts) Clase C: 192.0.0.0 - 223.255.255.255 (/24, 254 hosts) # Direcciones privadas (RFC 1918) 10.0.0.0 - 10.255.255.255 (10.0.0.0/8) 172.16.0.0 - 172.31.255.255 (172.16.0.0/12) 192.168.0.0 - 192.168.255.255 (192.168.0.0/16) # Direcciones especiales 127.0.0.1 Localhost (loopback) 0.0.0.0 Todas las interfaces / ruta por defecto 255.255.255.255 Broadcast limitado 169.254.x.x APIPA (sin DHCP)

    Máscara de Subred

    # La máscara determina qué parte es red y qué parte es host 255.255.255.0 = /24 (CIDR notation) 11111111.11111111.11111111.00000000 Red: primeros 24 bits Host: últimos 8 bits = 256 direcciones (254 usables) # Ejemplos de subredes /24 = 255.255.255.0 = 254 hosts /25 = 255.255.255.128 = 126 hosts /26 = 255.255.255.192 = 62 hosts /27 = 255.255.255.224 = 30 hosts /28 = 255.255.255.240 = 14 hosts /29 = 255.255.255.248 = 6 hosts /30 = 255.255.255.252 = 2 hosts (punto a punto) # Calcular rango de una subred IP: 192.168.1.100/26 Máscara: 255.255.255.192 Tamaño bloque: 256 - 192 = 64 Bloques: 0, 64, 128, 192 100 está en bloque 64 Red: 192.168.1.64 Broadcast: 192.168.1.127 Rango usable: 192.168.1.65 - 192.168.1.126 Gateway típico: 192.168.1.65

    Copias de Seguridad

    Tipos de Backup

    Completo (Full): - Copia todos los datos - Más tiempo y espacio - Restauración más rápida - Ej: Domingo → Full Incremental: - Solo cambios desde último backup (cualquier tipo) - Rápido y poco espacio - Restauración más lenta (necesita toda la cadena) - Ej: Lunes-Sábado → Incrementales Diferencial: - Cambios desde último Full - Equilibrio tiempo/espacio - Restauración: Full + último diferencial - Ej: Lunes-Sábado → Diferenciales

    Estrategia 3-2-1

    La regla 3-2-1 es el estándar para backups: 3 - Tres copias de los datos (original + 2 backups) 2 - En dos tipos de medios diferentes (disco local + nube) 1 - Una copia offsite (fuera del edificio) Ejemplo práctico: - Original: disco del equipo - Backup 1: NAS local (diario) - Backup 2: Cloud (semanal) → Offsite

Probar desde otra máquina curl -I http://IP_SERVIDOR # Problemas comunes: # - Puerto ocupado: lsof -i :80 # - Permisos de archivos: ls -la /var/www/html # - SELinux (en RHEL): getenforce, setenforce 0
