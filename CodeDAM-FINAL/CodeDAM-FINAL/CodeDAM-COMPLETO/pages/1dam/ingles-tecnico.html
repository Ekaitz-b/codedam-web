1. Working in IT

Professional Roles in IT

Software Developer / Programmer

Definition: A professional who writes, tests, and maintains code to create software applications.

Example: "As a software developer, I work on mobile applications using Java and Kotlin."

System Administrator (Sysadmin)

Definition: A professional responsible for managing and maintaining computer systems and servers.

Example: "The system administrator monitors server performance and handles security updates."

Database Administrator (DBA)

Definition: A professional who manages, maintains, and secures databases.

Example: "The DBA is responsible for database backups and query optimization."

Network Engineer

Definition: A professional who designs, implements, and maintains network infrastructure.

Example: "Our network engineer configured the new firewall for improved security."

DevOps Engineer

Definition: A professional who combines development and operations to improve deployment processes.

Example: "The DevOps engineer automated our deployment pipeline using Docker and Kubernetes."

Full-stack Developer

Definition: A developer who works on both front-end and back-end of applications.

Example: "As a full-stack developer, I handle everything from database design to user interface."

Front-end Developer

Definition: A developer who focuses on the user interface and user experience.

Example: "Front-end developers use HTML, CSS, and JavaScript to build responsive websites."

Back-end Developer

Definition: A developer who works on server-side logic, databases, and application architecture.

Example: "Back-end developers create APIs that front-end applications consume."

QA Tester / Quality Assurance

Definition: A professional who tests software to find and report bugs.

Example: "QA testers ensure the application works correctly before release."

Technical Support / Help Desk

Definition: A professional who assists users with technical problems.

Example: "Technical support helps customers troubleshoot software issues."

2. Computer Hardware

Main Components

CPU (Central Processing Unit)

Definition: The "brain" of the computer that processes instructions and performs calculations.

Also called: Processor, chip

Example: "This computer has an Intel Core i7 CPU running at 3.5 GHz."

Specifications: Clock speed (GHz), cores, threads, cache

Motherboard

Definition: The main circuit board that connects all components together.

Also called: Mainboard, system board

Example: "The motherboard provides slots for RAM, graphics card, and storage devices."

Features: Chipset, sockets, expansion slots

RAM (Random Access Memory)

Definition: Temporary memory used to store data that is actively being used.

Also called: Memory, system memory

Example: "You need at least 8 GB of RAM for modern software development."

Types: DDR4, DDR5

ROM (Read-Only Memory)

Definition: Permanent memory that stores firmware and boot instructions.

Example: "The BIOS is stored in ROM and initializes hardware during startup."

Hard Drive (HDD)

Definition: A storage device that uses magnetic disks to store data permanently.

Also called: Hard disk, HDD

Example: "This computer has a 1 TB hard drive for storing files and programs."

Specifications: Capacity (TB), speed (RPM), interface (SATA)

SSD (Solid State Drive)

Definition: A fast storage device that uses flash memory instead of spinning disks.

Example: "SSDs are much faster than traditional hard drives for boot times and application loading."

Types: SATA SSD, NVMe SSD, M.2

GPU (Graphics Processing Unit)

Definition: A specialized processor designed for rendering graphics and parallel processing.

Also called: Graphics card, video card

Example: "A powerful GPU is essential for gaming, video editing, and machine learning."

Brands: NVIDIA, AMD

Power Supply Unit (PSU)

Definition: A device that converts AC power to DC power for computer components.

Example: "Make sure your power supply has enough wattage for all your components."

Specifications: Wattage (W), efficiency rating (80+ Bronze, Gold, Platinum)

Cooling System

Definition: Components that dissipate heat from the CPU and other parts.

Components: Fans, heat sinks, thermal paste, liquid cooling

Example: "Proper cooling prevents overheating and extends hardware lifespan."

Case / Chassis

Definition: The enclosure that houses and protects all computer components.

Also called: Computer case, tower

Example: "Choose a case with good airflow and enough space for your components."

Types: Full tower, mid tower, mini tower, small form factor

Hardware Specifications Vocabulary

GHz (Gigahertz) - unit of CPU speed (billions of cycles per second)

GB (Gigabyte) - unit of memory/storage (1,024 MB)

TB (Terabyte) - unit of storage (1,024 GB)

cores - independent processing units within a CPU

threads - virtual cores that allow parallel processing

cache - fast memory inside the CPU

bandwidth - data transfer rate

latency - delay in data processing

Hardware Actions

assemble - build a computer from components

install - add a new component

upgrade - replace with a better component

replace - swap a faulty component

connect - attach cables or devices

disconnect - remove cables or devices

mount - secure a component in place

overclock - increase component speed beyond default

5. Programming & Development Fundamentals

Basic Programming Concepts

Variable

Definition: A container for storing data values.

Example: "A variable can hold different types of data like numbers or text."

Code example:

int age = 25;          // Integer variable
String name = "John";  // String variable

Function / Method

Definition: A reusable block of code that performs a specific task.

Example: "Functions help organize code and avoid repetition."

Code example:

// Function to calculate sum
public int calculateSum(int a, int b) {
    return a + b;
}

Loop

Definition: A structure that repeats a block of code multiple times.

Types: for loop, while loop, do-while loop

Example: "Use a loop to iterate through an array."

Code example:

// For loop example
for (int i = 0; i < 10; i++) {
    System.out.println(i);  // Print numbers 0 to 9
}

Array

Definition: A collection of elements of the same type stored in contiguous memory.

Example: "Arrays allow you to store multiple values in a single variable."

Code example:

// Declare and initialize an array
int[] numbers = {1, 2, 3, 4, 5};
System.out.println(numbers[0]);  // Access first element (1)

Condition / If Statement

Definition: A structure that executes code based on whether a condition is true or false.

Example: "Use if statements to make decisions in your code."

Code example:

if (age >= 18) {
    System.out.println("You are an adult");
} else {
    System.out.println("You are a minor");
}

Object-Oriented Programming (OOP)

Class

Definition: A blueprint or template for creating objects.

Example: "A class defines the properties and behaviors of objects."

Code example:

// Define a class
public class Person {
    String name;
    int age;
    
    // Constructor
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
}

Object

Definition: An instance of a class with its own data and behavior.

Example: "Objects are created from classes."

Code example:

// Create an object
Person john = new Person("John", 25);
System.out.println(john.name);  // Output: John

Method

Definition: A function defined inside a class.

Example: "Methods define the behavior of objects."

Code example:

public class Calculator {
    // Method to add two numbers
    public int add(int a, int b) {
        return a + b;
    }
}

Parameter / Argument

Definition: Values passed to a function or method.

Difference: Parameters are in the function definition; arguments are the actual values passed.

Example: "Functions can accept parameters to work with different data."

Return Value

Definition: The value that a function gives back after execution.

Example: "The function returns the calculated result."

Code example:

public int multiply(int a, int b) {
    return a * b;  // Return the product
}

/**
 * Validates user input for registration
 * @param username The username to validate
 * @param email The email address to validate
 * @return true if valid, false otherwise
 */
public boolean validateInput(String username, String email) {
    // Check if username is not empty and has valid length
    if (username == null || username.length() < 3) {
        return false;
    }
    // Validate email format
    return email.matches("^[A-Za-z0-9+_.-]+@(.+)$");
}

Naming Conventions

camelCase - first word lowercase, rest capitalized (myVariable, userName)

PascalCase - all words capitalized (MyClass, UserProfile)

snake_case - words separated by underscores (my_variable, user_name)

UPPER_CASE - all capitals with underscores (MAX_SIZE, API_KEY)

Convention Examples in Java

// Classes use PascalCase
public class StudentManager { }

// Variables and methods use camelCase
int studentCount = 0;
public void calculateAverage() { }

// Constants use UPPER_CASE
public static final int MAX_STUDENTS = 100;

// Packages use lowercase
package com.company.project;
